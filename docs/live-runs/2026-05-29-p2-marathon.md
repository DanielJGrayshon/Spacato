## Anomalies / observations worth recording (per §10.4 "future spec calibrations")

1. **Spec §12 forecast 28 layer-3 calls, run produced 30.** Calendar emitted 5 weeks per month for all six months (no 4-week months in the 6-month window starting 2026-05-29). The cost projection in §12 is therefore mildly under, but still well inside the user-facing "≈ $0.013" claim. Worth nudging §12 to say "~30 calls" or "28–30 calls".
2. **30-minute median is a strong attractor.** 103 / 185 dailies (55.7 %) land in (15, 30] — every rest/recovery/reflect day defaults to exactly 30 minutes, plus the early-month base runs. The output is coherent but slightly bimodal. Not a problem at v1 (P4's packer cares about real wall time, not LLM-suggested minutes), but a calibration note for P3 if reweighting uses minutes-as-cost.
3. **Light "reflective" / journaling days.** Tasks like "Rest and Reflect" (M1 W1 D5) and "Weekly Recap and Stretching" (M1 W1 D7) drift slightly toward mindfulness over training. Not a violation of any §10.4 criterion, but operator may want to consider tightening the layer-3 system prompt's framing if marathon coaches reading this would flinch.
4. **30 parallel layer-3 calls, no rate-limit hits.** `bypassCache: true` is doing its job (every weekly produced distinct daily content). The lack of rate limiting validates §3's "no throttle needed" claim — `Promise.all` fires all 30 simultaneously without any incident at `gpt-4o-mini`.
